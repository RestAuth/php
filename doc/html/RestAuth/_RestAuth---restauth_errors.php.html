Classes
Description | Classes
Class Description
RestAuthException Common superclass for all RestAuth related exceptions.
RestAuthResourceNotFound Superclass for exceptions thrown when a resource queried is not found.
RestAuthResourceConflict Superclass of exceptions thrown when a resource is supposed to be created but already exists.
RestAuthBadResponse Exception thrown when a response was unparsable.
RestAuthInternalException Superclass for service-related errors.
RestAuthBadRequest Thrown when the RestAuth service cannot parse the HTTP request. On a protocol level, this corresponds to a HTTP status code 400.
RestAuthInternalServerError Thrown when the RestAuth service suffers an internal error. On a protocol level, this corresponds to a HTTP status code 500.
RestAuthUnknownStatus Thrown when an unknown HTTP status code is encountered. This should never really happen and usually indicates a bug in the library.
RestAuthPreconditionFailed Thrown when you send unacceptable data to the RestAuth service, i.e. a password that is too short.
RestAuthUnauthorized Thrown when the user/password does not match the registered service.
RestAuthNotAcceptable Thrown when the RestAuth server cannot generate a response in the requested format.
RestAuthUnsupportedMediaType Thrown when the RestAuth server does not understand the content-type sent by this library.
RestAuthHttpException Thrown when a connection-related error occurs (i.e. the RestAuth service is not available).
