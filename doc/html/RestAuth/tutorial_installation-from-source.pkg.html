Installation from source

How-to install from source

Mathias Ertl
mati@restauth.net

Requirements

php-restauth requires few external libraries:

  • PHP 5.1 or later
  • pecl_http
  • PHPUnit if you want to run the test-suite
  • Xdebug if you want to create a test coverage
  • phpDocumentor if you want to generate source code documentation (this page)

Get source

From git

This project is developed on git.fsinf.at. You can view the source code at git.fsinf.at/restauth/php. To clone the repository to a directory named "php-restauth", simply do:

git clone http://git.fsinf.at/restauth/php.git restauth-php

Older versions are marked as tags. You can view available tags with git tag -l. You can use any of those versions with git checkout, for example git checkout 1.0. To move back to the newest version, use git checkout master.

Official releases

You can download official releases of php-restauth at https://php.restauth.net/download.

Installation

Install dependencies

You can use PECL to install pecl_http:

pecl install pecl_http

If you also want to run the test-suite, generate test coverage or generate source code documentation, you will need install the required packages as well.

Install php-restauth

Installation itself is very easy. Just go to the directory where your source is located ("php-restauth" in the above example) and just run:

pear install -f package.xml

On most systems, the pear install needs to run with superuser privileges.

You can verify that the installation worked by running this command from your home directory:

php -r "include('RestAuth/restauth.php');"

This will throw an error if php-restauth was not installed successfully.

Updating

If you ever want to update the source code, just use:

make clean; git pull; pear upgrade -f package.xml;

Run tests

Running the test-suite or generating a test coverage report will remove all data from the running RestAuth server. Do not perform these tests on a live installation.

RestAuthClient features an extensive test suite using phpunit. Since it implements a network protocol, the library requires a RestAuth server to run on http://[::1]:8000 that has the service vowi with the password vowi preconfigured. The RestAuth server contains the script test.sh in its source code to simply start a server and add the expected service.

Once you started the server, you can run the test-suite using:

make test

Test coverage

You can also generate a test coverage report by simply executing the command:

make coverage

The test coverage in HTML format will go to doc/coverage/.

Build documentation

To generate the most recent documentation (the newest version of the document you're currently reading), just run:

make doc
